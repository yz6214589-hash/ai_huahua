# 引导问题话术库

业务人员通常不善于结构化表达。本文档提供每一步的标准话术和追问策略。

## Step 1: 场景定位 -- 话术

### 开场白

"接下来我会通过几个简单的问题，帮你把这个经验整理成一个可复用的技能。每步只需要回答 2-3 个问题，我们开始吧。"

### 标准提问

"请回答以下 3 个问题:

1. **你在做什么？** 用一句话描述这个任务（比如：帮客户评估某只债券的信用风险）
2. **什么情况下会做这件事？** 列 1-3 个触发场景（比如：客户主动问、新债上市时、评级变动时）
3. **输入和输出是什么？** 你拿到什么信息，最终产出什么？（比如：拿到公司名称和债券代码，产出一段信用评估结论）"

### 追问策略

| 用户回答 | 追问话术 |
|---------|---------|
| 太宽泛，如"帮客户做分析" | "做什么类型的分析？能举一个最常见的例子吗？" |
| 只说了做什么，没说什么时候 | "一般什么情况下你会做这件事？是客户来问，还是定期做，还是有什么事件触发？" |
| 没说清输入输出 | "做这件事的时候，你手上通常已经有什么信息？最终要交出去的是什么形式的东西？" |

## Step 2: 示例采集 -- 话术

### 标准提问

"现在请给我讲一个你实际做过的案例，越具体越好:

1. 当时的具体输入是什么？（比如：拿到了 XX 公司 2025 年年报）
2. 你做了哪些步骤？一步一步说
3. 每一步参考了什么资料或系统？
4. 最终输出是什么样的？能给一段示例内容吗？"

### 追问策略

这是最容易出现"太简略"的环节，需要重点追问。

| 用户回答 | 追问话术 |
|---------|---------|
| "我会分析一下" | "分析的时候具体看哪些指标或内容？怎么判断好还是不好？" |
| "查一下数据" | "从哪个系统查？查哪些字段或指标？" |
| "对比一下" | "跟什么对比？上一期？同行？对比哪些维度？" |
| "写个结论" | "结论一般包含哪几部分？长度大概多少？能给一个例子吗？" |
| "看情况而定" | "能举两种不同情况的例子吗？分别是怎么处理的？" |
| 步骤之间跳跃太大 | "在 XX 和 YY 之间，有没有其他中间步骤？比如需要验证什么，或者转换什么格式？" |
| 只说了一个案例且很简单 | "还有没有一个稍微复杂一点的案例？或者遇到过什么特殊情况？" |

### 示例引导

如果用户完全不知道怎么说，可以给一个完整示例作为参考:

"比如，一个'债券信用评估'的案例可能是这样的:

输入: 客户问'XX 城投债能不能买'
步骤:
1. 先查 XX 公司的最新信用评级（中诚信、联合资信等）
2. 下载最近一期年报，看资产负债率、经营性现金流
3. 查看该地区的财政收入和债务率数据
4. 和同评级同地区的城投做横向对比
5. 综合判断，写出评估结论

输出: '建议关注/建议规避 + 理由 + 主要风险点'

你的案例也可以按这个格式来说。"

## Step 3: 规则提炼 -- 话术

### 标准提问

"最后几个问题，帮我提炼一下关键规则:

1. **必须做的事**: 做这个任务有什么不能跳过的步骤或者必须遵守的原则？
2. **绝对不能做的事**: 有什么常见的错误或者禁区？做了会出大问题的？
3. **参考文档**: 做这个任务时你会参考哪些文件、手册、系统或数据源？"

### 追问策略

| 用户回答 | 追问话术 |
|---------|---------|
| "没什么特别的" | "我注意到你在案例中提到了 XX（从 Step 2 提取），这是不是每次都必须做的？" |
| 只说了"必须"没说"不能" | "有没有什么常见的误区？新人刚开始做的时候容易犯什么错？" |
| 参考文档为空 | "做的时候完全凭经验？不需要查任何系统或文件？" |
| 规则太多太细 | "如果只能保留最重要的 3 条，你会选哪 3 条？" |

## Step 4: 生成确认 -- 话术

### 展示成果

"好的，我已经根据你的描述生成了技能文件。请看一下以下内容是否准确:

（展示生成的 SKILL.md 内容）

你觉得有什么需要修改的吗？比如:
- 步骤有遗漏或顺序不对？
- 规则需要补充或修改？
- 描述不够准确？"

### 确认完成

"技能已经创建完成，保存在 skills/{skill-name}/ 目录下。以后 Agent 遇到相关任务时就会自动参考这个技能来执行了。"
