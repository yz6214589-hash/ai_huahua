# 回测系统优化 - 项目计划

## 一、项目目标

将当前回测系统从"基础单股回测"升级为"专业量化回测验证框架"，核心目标：

1. **科学性**：引入训练/验证/测试区间划分和滚动验证，防止过拟合
2. **完整性**：添加基准对比、交易成本、风险指标，全面评估策略表现
3. **可视化**：提供净值曲线、回撤曲线、月度收益等专业图表
4. **可管理**：支持回测历史持久化、对比、导出

## 二、项目范围

### 包含（In Scope）
- 回测时间区间管理（训练/验证/测试划分）
- 基准指数对比（沪深300等）与Alpha/Beta/IR计算
- 交易成本配置（滑点、手续费、印花税）
- 回测指标增强（Calmar、Sortino、波动率等）
- ECharts可视化图表（净值曲线、回撤曲线、月度热力图）
- 滚动验证（Walk Forward Analysis）
- 回测历史持久化与对比
- 参数网格搜索

### 不包含（Out of Scope）
- 实盘交易对接（属于执行模块）
- 实时行情推送
- 策略自动生成（AI生成策略）
- 多资产类别支持（期货/期权/债券）

### 沟通机制
- 后端完成后提供API接口文档（请求/响应格式）
- 前端基于API文档开发，不依赖后端运行状态
- 每阶段结束由test-pro统一验收

## 五、风险评估与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 基准指数数据缺失 | 高 | 高 | 优先使用trade_stock_daily表中已有的指数代码；若无则提供自定义基准上传 |
| Backtrader滑点支持 | 中 | 中 | 使用bt.slippage.PercentSlippage或固定滑点类 |
| ECharts大数据量渲染慢 | 中 | 低 | 净值数据降采样（超过500点时采样） |
| 滚动验证耗时过长 | 高 | 中 | 使用异步任务+进度回调；限制最大滚动期数 |
| 向后兼容性破坏 | 低 | 高 | 保持原有API接口不变，新增字段均为可选 |
| 中文乱码 | 中 | 中 | 所有文件强制UTF-8编码，响应头设置charset |

## 六、验收标准

按项目规则（.trae/rules/开发验证标准.md）：
1. 前端页面正常打开，无 "Not Found"
2. API返回2xx状态码
3. 浏览器控制台无错误日志
4. 中文内容无乱码
5. 原有单股回测功能不受影响
6. 新增功能可正常操作和展示
