## 四、详细表结构分析

### 4.1 现有表详细结构

#### 1. trade_sentiment_run（舆情监控运行记录表）

**现有字段**：
```sql
run_id             VARCHAR(32)      -- 运行ID（主键）
trigger_type       VARCHAR(16)      -- 触发类型
stock_codes_json   TEXT            -- 股票代码JSON数组
stock_names_json   TEXT            -- 股票名称JSON数组
days              INT              -- 查询历史天数（默认3）
use_llm           TINYINT(1)      -- 是否启用LLM精检
status            VARCHAR(16)      -- 状态：waiting/running/success/failed
total_events      INT              -- 检测到的事件总数
created_at        DATETIME         -- 创建时间
started_at        DATETIME         -- 开始时间
finished_at       DATETIME         -- 完成时间
error_message     TEXT             -- 错误信息
```

**需要扩充的字段**：
```sql
schedule_id        INT              -- 关联的定时任务ID（新增）
positive_count     INT              -- 正面舆情数量（新增）
negative_count     INT              -- 负面舆情数量（新增）
neutral_count      INT              -- 中性舆情数量（新增）
total_news         INT              -- 新闻总数（新增）
```

**扩充字段的 SQL**：
```sql
ALTER TABLE trade_sentiment_run
ADD COLUMN schedule_id INT COMMENT '关联的定时任务ID' AFTER status,
ADD COLUMN positive_count INT NOT NULL DEFAULT 0 COMMENT '正面舆情数量' AFTER total_events,
ADD COLUMN negative_count INT NOT NULL DEFAULT 0 COMMENT '负面舆情数量' AFTER positive_count,
ADD COLUMN neutral_count INT NOT NULL DEFAULT 0 COMMENT '中性舆情数量' AFTER negative_count,
ADD COLUMN total_news INT NOT NULL DEFAULT 0 COMMENT '新闻总数' AFTER neutral_count;
```
