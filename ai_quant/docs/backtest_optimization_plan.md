## 三、分阶段优化方案

### 阶段一：核心回测功能增强（P0 优先级）

#### 目标：建立科学的回测验证框架，添加基准对比，完善交易配置

**功能1：训练/验证/测试区间划分**
- 支持用户设置三个独立时间区间
- 自动分配：训练70%，验证15%，测试15%（可配置）
- 区间可手动调整，有可视化预览

**功能2：基准指数对比**
- 支持沪深300、中证500、创业板指等主流指数
- 计算超额收益（Alpha）
- 净值曲线同时显示策略与基准

**功能3：完善的交易成本配置**
- 可配置的滑点（固定点数或百分比）
- 可配置的佣金率（买入/卖出分开）
- 支持印花税（仅卖出时）

**功能4：回测指标增强**
- 信息比率（IR）
- Beta系数
- 波动率
- Calmar比率
- 最大回撤周期（恢复时间）
- 交易次数分布（多/空，盈利/亏损）

### 阶段二：高级回测与可视化（P1 优先级）

#### 目标：增强回测分析能力，提供丰富的可视化图表

**功能5：滚动验证（Walk Forward Analysis）**
- 支持固定窗口滚动或展开窗口滚动
- 自动多期验证结果聚合
- 滚动稳定性统计

**功能6：丰富的可视化图表**
- 净值&基准对比曲线（使用ECharts）
- 回撤曲线
- 月度收益柱状图/热力图
- 盈亏分布直方图
- 逐笔盈亏时序图

**功能7：回测历史管理**
- 回测结果持久化到数据库
- 历史回测列表查询
- 多次回测对比展示

**功能8：高级统计指标**
- 持仓时间分布
- 换手率统计（日/周/月换手率）
- 最大连续盈利/亏损天数
- 盈亏比（Profit Factor）
- K比率

### 阶段三：参数优化与报告（P2 优先级）

#### 目标：支持策略参数优化，生成专业回测报告

**功能9：参数网格搜索**
- 单参数或多参数范围扫描
- 参数敏感性分析热力图
- Pareto最优解展示

**功能10：回测报告导出**
- PDF报告导出
- Excel详细数据导出
- 包含所有图表和指标

**功能11：策略持仓分析**
- 行业分布饼图
- 风格因子暴露（价值/成长/动量等）
- 换手率时序图

## 四、详细需求设计

### 4.1 回测时间区间管理

**设计目标**：防止过拟合，确保策略样本外表现

**功能描述**：
1. **区间自动划分**
   - 用户设置总回测范围（如2018-2025）
   - 系统自动按比例划分：训练70%，验证15%，测试15%
   - 例如：2018-2023训练，2024验证，2025测试

2. **区间手动调整**
   - 用户可拖拽或手动输入各区间日期
   - 可视化时间轴展示

3. **区间性能对比展示**
   - 训练集、验证集、测试集各指标并列展示
   - 红色警示样本内/外表现差异过大

**数据模型**：
```typescript
interface BacktestIntervalConfig {
  total_start: string
  total_end: string
  train_ratio: number      // 0.7
  val_ratio: number        // 0.15
  test_ratio: number       // 0.15
  custom_ranges?: {
    train: { start: string; end: string }
    val: { start: string; end: string }
    test: { start: string; end: string }
  }
}

interface IntervalPerformance {
  interval: 'train' | 'val' | 'test'
  start_date: string
  end_date: string
  metrics: BacktestMetrics
}
```

### 4.2 基准对比与Alpha计算

**功能描述**：
1. **基准选择**
   - 预设主流指数：沪深300、中证500、创业板指、上证50、中证1000
   - 支持自定义基准（代码输入）

2. **基准相关指标**
   - 超额收益（策略收益 - 基准收益）
   - Alpha（CAPM模型）
   - Beta系数
   - 跟踪误差（Tracking Error）
   - 信息比率（IR = Alpha / Tracking Error）

**前端展示**：
- 双Y轴：左侧净值，右侧回撤
- 策略曲线和基准曲线叠加对比
- 各区间分别计算基准对比指标

### 4.4 滚动验证（Walk Forward）

**设计理念**：
滚动验证是比单次样本外更科学的验证方法，模拟真实的策略迭代场景。

**两种滚动模式**：

| 模式 | 说明 |
|------|------|
| **固定窗口** | 每次用过去N年训练，接下来M年验证/测试，窗口不断滑动 |
| **展开窗口** | 训练集不断扩大，验证/测试窗口固定 |

**UI设计**：
```
滚动验证配置
────────────────
滚动模式: [固定窗口 ▼]
训练窗口: [5] 年
验证窗口: [1] 年
步长:     [1] 年

[开始滚动验证]
```

**结果展示**：
- 每期验证结果列表
- 滚动稳定性统计（各期指标均值、标准差、最大值、最小值）
- 收益分布箱线图

### 4.5 可视化图表设计

**前端库选型**：继续使用 ECharts（项目已有）

**图表清单**：

| 2. 回撤曲线 | 结果页 | 最大回撤标注 |
| 3. 月度收益热力图 | 分析页 | 颜色表示正负/大小 |

## 七、技术风险与注意事项

1. **过拟合检测**：区间功能核心目的，需确保训练集和测试集严格隔离
2. **计算性能**：滚动验证和参数搜索可能耗时，需要异步任务机制
3. **数据完整性**：基准指数数据需确保在回测期间有完整数据
4. **向后兼容**：现有单股票回测功能需保持完全兼容
