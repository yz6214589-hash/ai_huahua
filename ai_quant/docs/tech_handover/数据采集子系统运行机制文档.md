# 数据采集子系统完整运行机制文档

## 一、整体架构概览

数据采集子系统由以下核心组件构成：

| 组件 | 代码路径 | 关键行号 | 功能描述 |
|------|---------|---------|---------|
| API调度层 | `backend/api/jobs.py` | L25-L608 | FastAPI路由注册、定时调度(APScheduler)、并发控制 |
| 任务路由层 | `backend/core/jobs/runner.py` | L18-L39 | 统一入口`run_domain()`分发到9个domain任务 |
| 领域任务实现 | `backend/core/jobs/domains/*.py` | 各文件 | 9个独立采集任务的实现代码 |
| 公共数据模型 | `backend/core/jobs/common.py` | L10-L17 | JobStats数据类定义 |
| 数据库操作层 | `backend/core/db.py` | L17-L35(L58-L95) | MySQLConfig数据类、配置加载、连接管理 |
| 任务记录存储 | `backend/core/data/__init__.py` | L63-L75(L93-L129) | JSON文件持久化任务运行记录 |
| QMT网关客户端 | `backend/infra/qmt_gateway_client.py` | 全文件 L1-L306 | 远程QMT Gateway HTTP客户端(含自动重试) |

### 1.1 数据流

```
用户HTTP请求 / 定时器触发
    |
    v
api/jobs.py:run_job_once() -> 创建运行记录(JSON) -> BackgroundTasks异步
    |
    v
api/jobs.py:_run_job_impl() -> 调用 runner.run_domain()
    |
    v
core/jobs/runner.py:run_domain() -> 路由到具体 domain 函数
    |
    v
domains/*.py:run_xxx() -> 采集数据 -> executemany写入MySQL
    |
    v
更新运行记录JSON文件 (running -> success/partial/failed)
```

## 二、公共基础设施

### 2.1 任务统计数据结构

- **文件**: `backend/core/jobs/common.py` L10-L17
- 定义`JobStats`数据类，包含:
  - `items_processed` (L12): 处理的实体数量
  - `rows_written` (L13): 写入数据库的行数
  - `failed_items` (L14): 失败项目列表
  - `data_source_final` (L15): 最终使用的数据源
  - `fallback_chain` (L16): 数据源回退链路记录
  - `message` (L17): 消息(可选)

### 2.2 工具函数

- `normalize_stock_code()` - `common.py` L24-L33: 标准化股票代码，如"600519" -> "600519.SH"
- `to_ymd()` - `common.py` L36-L46: 将各种日期格式转为YYYY-MM-DD字符串
- `safe_float()` - `common.py` L56-L65: 安全浮点数转换

### 2.3 数据库配置与连接

- `MySQLConfig` - `backend/core/db.py` L17-L35: 数据库配置数据类
- `load_mysql_config()` - `db.py` L58-L95: 从环境变量加载数据库配置
- `connect()` - 创建MySQL数据库连接
- `executemany()` - 批量执行INSERT语句

### 2.4 QMT Gateway HTTP客户端

- **文件**: `backend/infra/qmt_gateway_client.py`
- `request_json()` - L66-L110: 核心HTTP请求函数，带Token认证和自动重试
- `historical_kline()` - L279-L306: 获取历史K线数据
- `check_health()` - L112-L119: 检查Gateway健康状态
- 配置: 通过`.env`文件设置`AI_QUANT_QMT_GATEWAY_BASE`和`AI_QUANT_QMT_GATEWAY_TOKEN`
- 超时设置: `AI_QUANT_QMT_GATEWAY_TIMEOUT`(默认5秒), `AI_QUANT_QMT_GATEWAY_KLINE_TIMEOUT`(默认60秒)

## 三、API调度层

### 3.1 API路由注册

- **文件**: `backend/api/jobs.py` L25
- 前缀: `/api/v1/jobs`

### 3.2 任务域定义

- `_KNOWN_JOB_DOMAINS` - L28-L38: 9个支持的任务域
- `_DEFAULT_SCHEDULES` - L40-L50: 每个域的默认定时调度配置(Cron表达式)

### 3.3 API接口清单

| 接口路径 | 方法 | 函数名 | 行号 | 功能描述 |
|---------|------|--------|------|---------|
| `/api/v1/jobs/runs` | GET | `list_runs()` | L382-L429 | 查询任务执行历史，含超时检测(L413-L427) |
| `/api/v1/jobs/runs/{run_id}` | GET | `get_run()` | L432-L442 | 查看单个任务运行详情 |
| `/api/v1/jobs/runs` | POST | `write_run()` | L445-L465 | 手动创建任务运行记录 |
| `/api/v1/jobs/run` | POST | `run_job_once()` | L468-L494 | 单次运行指定domain任务(主要入口) |
| `/api/v1/jobs/domains` | GET | `list_domains()` | L497-L510 | 查询所有支持的任务域 |
| `/api/v1/jobs/schedules` | GET | `list_schedules()` | L513-L560 | 查询所有定时调度配置 |
| `/api/v1/jobs/schedules/{domain}` | PUT | `update_schedule()` | L563-L608 | 更新指定任务域的调度配置 |

### 3.4 手动触发任务流程

- **入口**: `run_job_once()` - `api/jobs.py` L468-L494
  - L475-L477: 校验domain是否在`_KNOWN_JOB_DOMAINS`中
  - L478: 解析mode参数(默认为"test")
  - L479: 解析params参数
  - L481-L489: 通过`write_job_run()`创建运行记录(状态=running)
  - L493: 通过`BackgroundTasks.add_task()`异步执行`_run_job_impl()`

### 3.5 后台执行引擎

- `_run_job_impl()` - `api/jobs.py` L321-L379
  - L329: 调用`run_domain()`执行实际任务
  - L330: 根据failed_items判断状态(success或partial)
  - L331-L347: 更新运行记录为完成状态，写入rowsWritten/itemsProcessed等
  - L356-L378: 异常处理，写入failed状态和错误信息

### 3.6 定时调度(APScheduler)

- **文件**: `backend/api/jobs.py`
- `init_jobs_scheduler()` - L273-L289: 初始化BackgroundScheduler
- `start_jobs_scheduler()` - L292-L307: 读取数据库调度配置并注册所有定时任务
- `stop_jobs_scheduler()` - L310-L318: 关闭调度器
- `_enqueue_domain()` - L235-L270: 定时器触发时的执行函数(含并发锁控制)
- `_reschedule_all()` - L176-L232: 重新加载所有定时任务配置

### 3.7 超时检测机制

- **位置**: `api/jobs.py` `list_runs()` L413-L427
- 超时时间: `AI_QUANT_JOB_RUN_TIMEOUT_SECONDS`环境变量(默认900秒/15分钟)
- L414: 检测`status == "running"`且没有finishedAt的记录
- L420-L421: 计算运行时长，超过阈值则自动标记为failed
- L422-L427: 设置失败状态并填写超时提示信息

### 3.8 并发控制

- **位置**: `api/jobs.py` L66
- 每个domain一个`threading.Lock`，存储在`_LOCKS`字典中
- `_enqueue_domain()` L243: 使用`lk.acquire(blocking=False)`非阻塞获取锁
- L244-L248: 若锁已被占用则记录警告并跳过该次执行

### 任务1: stock_daily（行情日线）

#### 一、运行逻辑

- **入口函数**: `run_stock_daily()` - `domains/stock_daily.py` L194-L313
- **触发方式**: 手动POST /api/v1/jobs/run + APScheduler定时(0 18 * * 1-5)
- **参数**: mode, params.test_stock, params.data_start(默认20230101), params.max_stocks

**完整函数调用链**:
```
api/jobs.py:run_job_once() L468-L494
  -> api/jobs.py:_run_job_impl() L321-L379 (L329: stats = run_domain(domain, mode, params))
    -> runner.py:run_domain() L21-L22 (d == "stock_daily" -> run_stock_daily())
      -> stock_daily.py:run_stock_daily() L194-L313
        -> _latest_dates(conn) L32-L39       # 查询各股票最新日期
        -> _get_stock_list() L48-L75         # 获取股票列表
        -> check_health() L223-L224          # QMT健康检查
        -> 循环每只股票(L233-L301):
          -> _fetch_qmt() L78-L104            # QMT优先
          -> _fetch_akshare() L107-L127       # akshare备用
          -> _fetch_tushare() L130-L149       # tushare三级备用
          -> _rsi14() L152-L174               # 计算RSI14
          -> _ma20() L177-L191                # 计算MA20
        -> executemany(conn, _INSERT_SQL, batch_rows) L303
```

#### 二、数据范围

- 时间范围: 从已采集的最新日期到当日，首次采集从`data_start`(L197, 默认20230101)开始
- 数据量: `max_stocks`(L198) test模式=1只, full模式=200只(可覆盖)
- 粒度: 日线，每股每天一条
- 业务范围: A股全市场(通过akshare `stock_info_a_code_name`获取约5500只)

#### 三、数据来源

| 数据源 | 调用函数 | 行号 | 优先级 | 访问方式 |
|--------|---------|------|--------|---------|
| QMT Gateway | `_fetch_qmt()` | L78-L104 | 1(最高) | `historical_kline()` HTTP POST到远程Gateway(60秒超时) |
| AkShare | `_fetch_akshare()` | L107-L127 | 2 | `ak.stock_zh_a_hist()` 东方财富接口 |
| TuShare | `_fetch_tushare()` | L130-L149 | 3 | `pro.daily()` 需TUSHARE_TOKEN |

#### 四、关键代码实现

- **股票列表获取** `_get_stock_list()` L48-L75: test模式(L49-L51)仅返回指定测试股票，full模式(L53-L75)通过akshare获取全市场列表
- **QMT数据获取** `_fetch_qmt()` L78-L104:
  - L80-L89: 调用QMT Gateway的historical_kline接口
  - L93-L101: 解析返回的DataFrame，转换为统一格式(date/close/volume)
- **RSI14计算** `_rsi14()` L152-L174:
  - L155: 需要至少15个数据点
  - L157-L159: 计算每日涨跌幅
  - L160-L161: 计算起始14日平均涨跌幅
  - L162-L173: 使用平滑公式递推后续RSI值
- **MA20计算** `_ma20()` L177-L191: 滑动窗口平均，L186-L187窗口满后每次移除最旧值
- **数据写入SQL** L19-L29: `INSERT ... ON DUPLICATE KEY UPDATE`

  - L321: 同样使用`_run_with_timeout()`防止卡死

- **数据库回退** `_get_existing_financial_data_from_db()` L362-L405:

  - L385-L401: 转换为统一格式
- **主循环** L465-L524:
  - L473: 先尝试QMT获取行情
  - L476-L482: QMT无PE时从批量akshare数据查
  - L490: 获取财务数据(优先akshare)
