3-grid_strategy_2025.py 代码逻辑梳理

标的：600519.SH（贵州茅台） 区间：2025-01-01 ~ 2025-12-31 策略：等差网格交易 初始资金：100 万 每次交易：100 股

1. 脚本目标

该脚本对单只股票执行“固定价位网格策略”回测，核心是利用震荡行情做高抛低吸：

  • 买入网格：[1450, 1400, 1350, 1300]，价格下穿对应网格时买入。
  • 卖出网格：[1550, 1600, 1650, 1700]，价格上穿对应网格时卖出。
  • 通过“层级 position_level”约束触发顺序，避免网格被重复错误触发。
  • 输出交易记录、净值曲线、汇总报告和图表。

3. 核心机制：层级驱动的网格触发

脚本最关键的设计是 position_level，它表示当前已买入了多少层网格：

  • 层级 0：空仓，只允许触发第 1 个买入网格（1450）。
  • 层级 1：可继续触发 1400 买入，也可触发 1550 卖出。
  • 层级 2：可触发 1350 买入，也可触发 1600 卖出。
  • 层级 3：可触发 1300 买入，也可触发 1650 卖出。
  • 层级 4：买入网格已满，只可触发 1700 卖出。

每次买入层级 +1，每次卖出层级 -1，保证“先买后卖、卖完再买”的顺序闭环。

4. 交易触发规则

4.1 买入触发

  • 前一日价格在网格上方，当前价格跌破或触及网格：prev > grid >= current。
  • 使用对应网格价作为成交价（限价成交近似）。
  • 现金足够覆盖买入金额 + 手续费才成交。
  • 成交后更新：现金减少、持股增加、层级 +1，并记录 trade。

4.2 卖出触发

  • 前一日价格在网格下方，当前价格突破或触及网格：prev < grid <= current。
  • 仅当当前持股不少于每次交易股数（100 股）才允许卖出。
  • 成交后更新：现金增加净额（扣手续费）、持股减少、层级 -1，并记录 trade。
  • 同一交易日内，代码优先检查买入，若已成交则不再执行卖出。

5. 主流程执行顺序

  1. 加载并校验 CSV 数据，筛选 2025 年回测区间。
  2. 初始化 GridStrategy（账户、网格线、层级状态）。
  3. 按交易日循环，传入当日价和前一日价执行网格触发。
  4. 每日收盘记录净值：nav = cash + shares * current_price。
  5. 回测结束计算总收益率、最大回撤、买卖次数、手续费。
  6. 打印交易明细，保存 CSV/TXT，绘制并保存结果图。

6. 输出产物

  • outputs/grid_strategy_2025_nav.csv：日期、净值、收益率。
  • outputs/grid_strategy_2025_trades.csv：逐笔网格交易（有交易时）。
  • outputs/grid_strategy_2025_summary.txt：回测汇总指标。
  • outputs/grid_strategy_2025_chart.png：股价+网格线+资金曲线图。

7. 结果统计口径

  • 总收益率：(期末净值 / 期初净值) - 1。
  • 最大回撤：净值相对历史峰值的最深跌幅。
  • 交易统计：总交易次数、买入次数、卖出次数、累计手续费。
  • 期末状态：期末持股与期末现金，可判断是否留仓。

8. 边界与健壮性

已实现的保护：
  • 数据文件不存在时直接提示并退出。
  • 缺少 close 列时退出。
  • 回测区间无数据时退出。
  • 账户资金不足或股票不足时不会强制成交。
  • 主流程有异常捕获，报错时打印 traceback。

9. 代码逻辑一句话总结

该脚本是一个“带层级约束的固定网格回测器”：在预设价位被穿越时按网格买卖，通过层级状态管理仓位节奏，最终输出净值、交易和可视化结果。
