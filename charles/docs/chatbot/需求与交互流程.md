## 1. 范围与核心价值

### 1.1 做什么

1) **对话式投研问答**（Chat UI + 后端 Agent）
- 用户输入自然语言问题，Agent 自动规划并调用工具（脚本）完成分析，最后输出结构化结论（仓位建议、风险提示、标的建议、后续深挖入口）。

2) **至少覆盖 5 类场景**（你给定的场景 1~5）
- 场景 1：宏观风险 + 个股舆情联动
- 场景 2：个股舆情分析（比亚迪）
- 场景 3：A股重大事件检测（资产重组/回购等）
- 场景 4：全球市场情绪与加仓判断
- 场景 5：全链路投资决策（宏观→预测→事件→财报RAG→五步法研报）

3) **工具链必须采用 DeepAgents**
- 以 @tool 函数暴露能力，Agent 通过 DeepAgents 调用工具。

### 1.2 不做什么（本期不强制）
- 不做登录体系/多租户/权限隔离（沿用现有最小化方式）。
- 不做“自动下单”或自动交易。
- 不做复杂会话记忆长期存储（可选：仅存最近 N 条到 DB）。
- 不做 iframe 内嵌第三方站点（Polymarket 仅外链）。

## 2. Agent 能力设计（产品口径）

### 2.1 Agent 输入输出

**输入**：
- 用户自然语言问题
- 可选上下文：当前页面选择的股票代码、用户手动指定股票列表、days 参数等

**输出（统一格式建议）**：
- 摘要结论（1~3 条）
- 量化/结构化结果（宏观指数、恐慌贪婪、事件数量、正负面占比）
- 仓位建议（例如：0~100% 建议区间）+ 操作策略（分步）
- 风险提示（必含）
- 证据引用（脚本输出中带的来源字段或链接）
- 后续深挖入口（如：一键生成研报任务 / 一键跑事件检测 / 一键打开某 Run 详情）

### 2.2 场景路由（Agent 内部规划策略）

Agent 不要求用户显式选择场景，由 Agent 识别意图并自动组合工具：

1) “现在适合买入A股吗”“仓位建议”“宏观风险” → 场景 1/4
2) “持有某股”“消息面如何”“舆情” → 场景 2
3) “有没有资产重组/回购”等关键词 → 场景 3
4) “完整投资决策”“从宏观到标的研究” → 场景 5

## 3. 工具（脚本）映射与执行规范

> 约束：Agent 通过 DeepAgents 工具来执行脚本，脚本输出需解析并纳入最终结论。

### 3.1 宏观恐慌指标（market_fear_index.py）

对应场景：1、4、5  
执行：`python skills/sentiment-analysis/scripts/market_fear_index.py --output_dir output`  
输出：VIX/OVX/GVZ/US10Y、综合恐慌/贪婪指数、阈值解读、建议

### 3.2 预测市场（polymarket_monitor.py）

对应场景：1、5  
执行：`python skills/sentiment-analysis/scripts/polymarket_monitor.py --output_dir output`  
输出：事件概率变化、聪明钱信号、资产配置提示、外链入口

### 3.3 A股热点新闻（news_fetcher.py）

对应场景：1、3、5  
执行（示例）：
- 热点：`python skills/sentiment-analysis/scripts/news_fetcher.py --days 3`
- 关键词：`python skills/sentiment-analysis/scripts/news_fetcher.py --keywords 资产重组,回购 --days 3`
- 个股：`python skills/sentiment-analysis/scripts/news_fetcher.py --stock 002594 --days 7`
输出：news_file 路径（JSON/CSV）或直接输出内容

### 3.4 情感分析（sentiment_scorer.py）

对应场景：1、2、5  
执行：`python skills/sentiment-analysis/scripts/sentiment_scorer.py --news_file 路径 --output_dir output`  
输出：整体情绪、正负面占比、恐惧贪婪映射、风险/机会提示

### 3.5 事件检测（event_detector.py）

对应场景：3、5  
执行：`python skills/sentiment-analysis/scripts/event_detector.py --news_file 路径 --output_dir output`  
输出：事件列表、信号映射、标的推荐

### 3.6 财报/研报 RAG（read-pdf）

对应场景：5  
执行：`python skills/read-pdf/scripts/query_report.py --index_dir data/vector_store --query ... --stock ...`  
输出：带页码溯源的引用片段

### 3.7 五步法研报（write-report）

对应场景：5  
执行：`python skills/write-report/scripts/report_generator.py ...` 或 five_step_analysis.py  
输出：Markdown 研报

## 6. 交互流程（你确认后再做 HTML）

### 6.1 打开对话机器人页（/assistant）
1) 默认展示欢迎卡片：
   - 提示可问的问题模板（对应场景 1~5）
   - 提示可选参数：股票代码、days、是否启用“深度模式（全链路）”
2) 用户输入问题并发送
3) 页面展示：
   - Assistant “分析中…”状态
   - （可选）展开“工具调用日志”（展示脚本名/参数/耗时/输出文件）
4) 返回最终回答，包含：
   - 结论 + 仓位建议
   - 风险提示
   - 引用与链接
   - 可点击的“后续动作按钮”（例如：打开宏观页 / 触发舆情 Run / 创建研报任务）

### 6.2 场景快捷发起
页面提供 5 个快捷按钮：
- 宏观风险+舆情联动
- 个股舆情（比亚迪）
- 重大事件检测
- 全球市场情绪
- 全链路投资决策

点击按钮会自动填充输入框模板（用户可修改再发送）。

## 7. 后端接口（先定口径，确认后实现）

### 7.1 对话接口

- `POST /api/assistant/chat`
  - 入参：
    - `message: string`
    - `session_id?: string`（可选）
    - `context?: { stock_codes?: string[], days?: number, mode?: "normal"|"deep" }`
  - 返回（非流式版本）：
    - `answer: string`（Markdown）
    - `tool_runs: [{name, args, status, output_preview, output_files}]`（可选）
    - `session_id: string`

（可选增强）流式：
- `POST /api/assistant/chat_stream`（SSE / chunked）

### 7.2 会话历史（可选）
- `GET /api/assistant/sessions`
- `GET /api/assistant/sessions/{session_id}`
- `DELETE /api/assistant/sessions/{session_id}`
