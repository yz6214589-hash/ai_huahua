## 1. 模块范围

### 1.1 做什么

1) **自选股定期舆情监控（核心闭环）**  
- 以“自选股列表”为监控对象（复用现有自选股模块）。  
- 周期性执行：抓取新闻/公告 → 情感分析 → 事件识别与分类 → 输出“策略建议”。  
- 输出示例（符合你给的范式）：  
  - 获取新闻 → 识别事件类型“股东减持” → 推断“短期负面压力” → 建议“回避或减仓”。  

1.1) **手动选择股票执行舆情分析（补充能力）**  
- 除了“自选股定期扫描”，支持手动选择 1~N 只股票，立即执行该股票（组）的新闻抓取、情感分析、事件识别与策略建议。  
- 典型场景：临时关注某股票（不加入自选股）但希望立刻获得舆情结论。  

2) **宏观指标展示（只展示，不做交易）**  
- VIX、OVX、GVZ、美国10年期国债收益率  
- Polymarket 跳转链接（可选：展示常用关键词入口，如 tariff/war/ceasefire/China）  

### 1.2 不做什么（本期明确不做）

- 不做自动下单/自动交易。  
- 不做覆盖全市场（仅对自选股）。  
- 不做复杂回测（后续可扩展）。  
- 不在本期强制接入“智能研报”的五步法深度生成（但保留“从事件一键跳转研报任务”的扩展点）。

## 2. 核心概念与数据结构（产品口径）

### 2.1 舆情监控任务（Run）

一次“定期扫描”的执行实例，建议包含：
- 触发方式：定时触发 / 手动触发  
- 覆盖范围：本次扫描的自选股快照  
- 时间：创建时间、开始时间、结束时间  
- 状态：等待、运行中、完成、失败  
- 结果摘要：本次扫描产出多少条事件、多少条负面/正面等  

### 2.2 个股舆情结果（Item）

一次 Run 对某一只股票的汇总结果，建议包含：
- 新闻抓取摘要：抓取条数、时间范围、关键词命中情况  
- 情感指数：整体情绪（贪婪/乐观/中性/谨慎/恐慌）+ Fear&Greed 0~100  
- 事件检测：事件列表（利好/利空/政策 → 具体事件类别）  
- 策略建议：按事件映射生成信号（看多/谨慎/回避/强烈回避…）  

### 2.3 宏观快照（Macro Snapshot）

一次宏观指标刷新结果，建议包含：
- VIX/OVX/GVZ/US10Y 最新值、更新时间、风险等级/解读  
- 综合恐慌贪婪指数（0~100）及建议  
- Polymarket 热门入口（外链）

## 3. 规则与策略（产品层明确口径）

### 3.1 舆情数据来源与优先级

按课程文档“三层结构”口径落地：
- 一手：公告（可靠性最高）  
- 二手：财经媒体新闻（分钟级）  
- 三手：社媒/股吧（噪音大，本期不接入）  

本期实现：**公告 + 东方财富新闻 +（可选）央视政策新闻**（对应 nanobot 的 `news_fetcher.py`）。  

### 3.2 事件分类体系（复用 nanobot）

事件类型：利好 / 利空 / 政策  
事件类别示例：
- 利好：资产重组、回购增持、业绩预增、股权激励、大额订单、分红送转  
- 利空：业绩预减、违规处罚、股东减持、商誉减值、退市风险、诉讼仲裁  
- 政策：货币政策、产业政策、监管政策  

策略建议：按 `SIGNAL_MAP` 映射输出（与 [event_detector.py](file:///Users/apple/Desktop/ai_huahua/week8/%E8%AF%BE%E7%A8%8B%E4%BB%A3%E7%A0%81-20260408/CASE-AI%E9%87%8F%E5%8C%96%E5%8A%A9%E6%89%8B%EF%BC%88nanobot%EF%BC%89/skills/sentiment-analysis/scripts/event_detector.py) 一致）。  

### 3.3 情感分析口径（复用 nanobot）

- 单条新闻：正面/负面/中性 + 强度 1~5 + 关键实体 + 关键词 + 一句话摘要 + 市场影响  
- 聚合结果：整体情绪 + Fear&Greed 0~100 + 风险提示 + 机会提示  
对应 [sentiment_scorer.py](file:///Users/apple/Desktop/ai_huahua/week8/%E8%AF%BE%E7%A8%8B%E4%BB%A3%E7%A0%81-20260408/CASE-AI%E9%87%8F%E5%8C%96%E5%8A%A9%E6%89%8B%EF%BC%88nanobot%EF%BC%89/skills/sentiment-analysis/scripts/sentiment_scorer.py)。  

### 3.4 宏观指标口径（复用 nanobot）

指标：VIX / OVX / GVZ / US10Y  
输出：风险等级、阈值解释、综合恐慌贪婪评分、风险传导判断（OVX vs VIX）  
对应 [market_fear_index.py](file:///Users/apple/Desktop/ai_huahua/week8/%E8%AF%BE%E7%A8%8B%E4%BB%A3%E7%A0%81-20260408/CASE-AI%E9%87%8F%E5%8C%96%E5%8A%A9%E6%89%8B%EF%BC%88nanobot%EF%BC%89/skills/sentiment-analysis/scripts/market_fear_index.py)。  

### 3.5 Polymarket 展示口径（复用 nanobot）

- 展示为外链入口（本期不在前端内嵌市场详情）  
- 可提供几个快捷入口（关键词 preset）：`war`/`ceasefire`/`tariff`/`China`/`Fed`  
对应 [polymarket_monitor.py](file:///Users/apple/Desktop/ai_huahua/week8/%E8%AF%BE%E7%A8%8B%E4%BB%A3%E7%A0%81-20260408/CASE-AI%E9%87%8F%E5%8C%96%E5%8A%A9%E6%89%8B%EF%BC%88nanobot%EF%BC%89/skills/sentiment-analysis/scripts/polymarket_monitor.py)。

## 5. 交互流程（你确认后下一步做 HTML）

### 5.1 舆情监控首页（/sentiment）

**区域 A：任务与调度（顶部卡片）**  
- 显示：  
  - 调度状态：已启用/未启用  
  - 扫描频率：默认“每日收盘后（15:10）”  
  - 最近一次运行：开始/结束时间、状态  
  - 下一次运行时间（如可计算）  
- 操作：  
  - 立即执行一次（手动触发 Run）  
  - 启用/暂停定时监控  
  - 配置频率（最小化配置：下拉选择“每日收盘后（15:10）”或其他固定频点）  

**区域 A2：手动分析股票（同页卡片）**  
- 股票搜索框（支持多选，复用 `/api/stocks?q=`）  
- 参数：最近 N 天（默认 3 天）、是否启用 LLM 精检（默认关闭，可切换）  
- 操作：立即分析所选股票（生成一次 Run，触发方式=手动，覆盖范围=所选股票）  
 - 展示位置：放在“事件列表”上方，便于先选股再立即触发分析  

**区域 B：本次（或最近一次）事件列表（表格）**  
列表字段（按你给的例子展开成可读列）：
- 股票代码/公司名称  
- 事件类型（利好/利空/政策）  
- 事件类别（如：股东减持）  
- 策略建议（如：谨慎/回避/看多）  
- 影响推断（短句，可由映射 reason + LLM摘要组合）  
- 置信度（1~5）/ 紧急度（高/中/低）  
- 触发时间（来自新闻发布时间或 Run 时间）  
- 操作：查看详情  

筛选项：
- 股票公司（下拉/输入框，复用 `/api/stocks?q=` 搜索）  
- 事件类型（利好/利空/政策）  
- 时间范围（最近 1 天/7 天/30 天 或 日期选择）  

**区域 C：任务运行记录（可折叠）**  
- 展示最近 N 次 Run：创建时间、状态、耗时、覆盖股票数、事件数、失败原因（如有）  
- 操作：查看本次 Run 的明细（跳转详情页）  

### 5.2 Run 详情页（/sentiment/runs/{run_id}）

页头：Run 基本信息（状态、耗时、覆盖股票、事件数）  

**Tab 1：个股汇总**  
- 每只股票一行：新闻条数、情绪指数（0~100）、整体情绪（贪婪/恐慌…）、事件数、最高优先级事件  
- 点击进入该股票的“舆情详情”抽屉/子页  

**Tab 2：事件明细**  
- 展示该 Run 的所有事件（同首页表格，但更完整）  

### 5.3 股票舆情详情（/sentiment/stocks/{code}?run_id=xxx）

内容分三块：
1) 新闻列表：标题/时间/来源/摘要（可折叠展示原文片段）  
2) 情感分析：单条新闻结果 + 聚合指数 + 风险点/机会点  
3) 事件识别：事件列表 + 对应策略建议（映射原因 + LLM补充一句“短期影响推断”）  

### 5.4 宏观风险页（/sentiment/macro 或首页 Tab）

展示卡片：
- VIX / OVX / GVZ / US10Y：数值、更新时间、风险等级/阈值解释  
- 综合恐慌贪婪指数（0~100）：整体情绪 + 建议  
- 风险传导：OVX 与 VIX 的组合判断（如果满足条件显示一条结论）  

Polymarket 区域：
- 展示若干快捷链接按钮（打开新标签页）：`war` / `ceasefire` / `tariff` / `China` / `Fed`  
- 也提供一个输入框：输入关键词后打开 Polymarket 搜索页（可选）

## 6. 后端接口（先定口径，确认后实现）

### 6.1 舆情监控（自选股）

- `POST /api/sentiment/runs`  
  - 入参：`{ days: number, include_cctv?: boolean, use_llm?: boolean }`（可先固定默认值）  
  - 返回：`{ run_id, status }`  

- `GET /api/sentiment/runs`  
  - query：`status? created_start? created_end? limit?`  
  - 返回：`{ runs: [...] }`  

- `GET /api/sentiment/runs/{run_id}`  
  - 返回：Run + items + events（或分页拆分）  

- `GET /api/sentiment/stocks/{code}`  
  - query：`run_id`  
  - 返回：该股票在该 Run 的新闻/情感/事件详情  

- `PUT /api/sentiment/schedule`  
  - 入参：`{ enabled: boolean, interval_minutes?: number }` 或 `{ cron: string }`（二选一，确认后定）  
  - 返回：`{ ok: true }`  

### 6.2 宏观指标

- `POST /api/macro/snapshot`（手动刷新一次）  
- `GET /api/macro/latest`（返回最近一次快照）

## 7. 定时任务行为（确认后落到实现）

触发策略（建议默认）：
- 默认：**每日收盘后（15:10）** 扫描一次自选股（更省资源）。  
- 手动触发：随时可发起一次 Run；支持两类入口：  
  - 手动触发“扫描自选股”  
  - 手动选择股票后“仅分析所选股票”  

执行流程（一次 Run）：
1) 拉取目标股票列表快照（代码+名称），来源=自选股 或 手动选择  
2) 对每只股票：执行新闻抓取（N天）  
3) 对抓取到的新闻：情感分析（LLM）  
4) 对新闻：事件识别（默认关键词；可切换“关键词 + LLM 精检”）  
5) 事件 → 策略建议（映射）  
6) 保存 Run/Item/Event；前端可实时看到状态变化

## 8. 已确认的关键点（最终口径）

1) 默认扫描频率：每日收盘后（15:10）

2) 新闻时间范围 days 默认值：3 天

3) 事件识别策略：默认只用关键词，但支持切换“关键词 + LLM 精检”

4) 页面结构：“舆情监控”一个页面，顶部 Tab（自选股舆情 / 宏观风险）

5) 补充：除了自选股外，还支持手动选择股票并立即执行舆情收集与分析
